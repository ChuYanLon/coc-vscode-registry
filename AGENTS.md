# AGENTS.md — coc-vscode-registry

Standalone repo (sibling of [coc-vscode-loader](https://github.com/coc-plugin/coc-vscode-loader)). Static GitHub Pages site — no build, no package.json, no npm.

## Structure

| File | Purpose |
|------|---------|
| `registry.json` | Extension manifest entries (edit this to add/change plugins) |
| `presets.json` | Bridge preset definitions (ts-bridge) |
| `repo-status.json` | Auto-generated metadata (stars, archived, lastUpdated) |
| `scripts/check-repo-status.ts` | Runs weekly via CI; needs `gh` CLI authenticated |

## Data files

- **Entry naming**: `vscode-<short-name>` (e.g. `vscode-clangd`)
- **`minPluginVersion`** semver constraints documented in `README.md` table (local servers → `1.4.2`, module args → `1.4.3`, patches → `1.4.5`, go/cargoPackages → `1.5.0`, targetAssets → `1.5.0`, etc.)
- **Convert step types**: `language-client`, `source`, `bridge`, `mark-unsupported`, `snippets`
- **Server source**: prefer `prebuilt` (`type: "vsix"`) over compiling `kind: "module"` servers that need tsc/wasm builds
- Source `patches`, `excludeDeps`, `keepDeps` are per-entry fields under the `source` step

## CI

- **Deploy** (`.github/workflows/deploy.yml`): pushes to `main` → GitHub Pages. Only triggers on changes to `registry.json`, `repo-status.json`, `index.html`, `script.js`, `style.css`, `assets/**`
- **Check Repo Status** (`.github/workflows/check-repo-status.yml`): weekly cron + manual dispatch. Runs `npx tsx scripts/check-repo-status.ts`, auto-PRs `repo-status.json` changes, creates issues for archived repos

## Commands

```bash
npx tsx scripts/check-repo-status.ts   # check GitHub repos via gh API
```

Requires `gh` CLI with auth.

## Style

`.editorconfig` only (2-space indent, LF, UTF-8, trim trailing whitespace). No linter, no formatter, no typecheck.

## Gotchas

- This repo holds **data only**. The converter/parser is in `coc-vscode-loader`. Adding a plugin requires changes in both repos.
- `repo-status.json` is auto-generated by CI — do not edit manually.
- Registry website is served at `coc-plugin.github.io/coc-vscode-registry/`.
- No local dev server; open `index.html` directly in a browser to preview.
